MPI

Message Passing Interface

  • Basado en memoria distribuida
  • SPMD
    • Single program
    • Multiple data
  • Proceso: programa corriendo en un par (CPU + memoria)
  • Comunicación y sincronización via send + receive

MPI es una API, definida de forma estándar para FORTRAN y C

Hola mundo, en MPI

Código fuente

 1 #include <stdio.h>
 2 #include <string.h>
 3 #include <mpi.h>
 4 
 5 const int MAX_STRING = 100;
 6 
 7 int main(int argc, char **argv) {
 8     char greeting[MAX_STRING];
 9     int comm_sz;
10     int my_rank;
11 
12     MPI_Init(&argc, &argv);
13     MPI_Comm_size(MPI_COMM_WORLD, &comm_sz);
14     MPI_Comm_rank(MPI_COMM_WORLD, &my_rank);
15 
16     if (my_rank != 0) {
17         sprintf(greeting, "Saludos desde el proceso %d de %d!", my_rank, comm_sz);
18         MPI_Send(greeting, strlen(greeting)+1, MPI_CHAR, 0, 0, MPI_COMM_WORLD);
19     } else {
20         int q;
21         printf("Saludos desde el proceso %d de %d!\n", my_rank, comm_sz);
22         for (q = 1; q < comm_sz; q++) {
23             MPI_Recv(greeting, MAX_STRING, MPI_CHAR, q, 0,
24                      MPI_COMM_WORLD, MPI_STATUS_IGNORE);
25             printf("%s\n", greeting);
26         }
27     }
28 
29     MPI_Finalize();
30     return 0;
31 }

SPMD

  • No se escriben programas separados por proceso
  • No necesariamente todos los procesos acaban haciendo lo mismo
    • distinto rank
    • distintos datos
  • En realidad eso es la situación típica
  • Diseño "escalable" de los programas

En el ejemplo, el proceso 0 recibe mensajes y los muestra

1         int q;
2         printf("Saludos desde el proceso %d de %d!\n", my_rank, comm_sz);
3         for (q = 1; q < comm_sz; q++) {
4             MPI_Recv(greeting, MAX_STRING, MPI_CHAR, q, 0,
5                      MPI_COMM_WORLD, MPI_STATUS_IGNORE);
6             printf("%s\n", greeting);
7         }

Los procesos [1..n) generan mensajes y los envían

1         sprintf(greeting, "Saludos desde el proceso %d de %d!", my_rank, comm_sz);
2         MPI_Send(greeting, strlen(greeting)+1, MPI_CHAR, 0, 0, MPI_COMM_WORLD);

15/36

Comunicación

Message matching

Si q envía, r puede recibir el mensaje cuando

  • recv_comm == send_comm
  • recv_tag == send_tag
  • dest == r
  • source == q

Además para que la operación sea exitosa, los buffers deben ser compatibles en tipo y tamaño.

Hay “comodines” en el receptor:

  • MPI_ANY_SOURCE
  • MPI_ANY_TAG

Ejemplo: regla del trapecio

Se aproxima una integral como suma de áreas de trapecios

Dividimos el intervalo [a,b] en n segmentos, cada uno de h=b-a/n de ancho. Esto define puntos x[0]=a, x[1], x[2], ..., x[n]=b. Tenemos que x[k+1] = x[k]+h.

Cada trapecio tiene área (f(x[i]) + f(x[i+1])) h / 2

Entonces nuestra aproximación es

I = h ( f(x[0])/2 + f(x[1]) + f(x[2]) + ... + f(x[n-1]) + f(x[n])/2)

19/36

Ejemplo: regla del trapecio

Versión secuencial

Sin paralelismo:

1 h = (b-a)/n;
2 approx = (f(a) + f(b))/2.0;
3 for (i = 1; i <= n-1; i++) {
4     x_i = a + i*h;
5     approx += f(x_i);
6 }
7 approx = h*approx;

Se paraleliza con una “receta” típica:

  1. Particionar la solución en tareas
  2. Identificar los canales de comunicación entre tareas
  3. Acumular las tareas en tareas mayores
  4. Asignar las tareas mayores a cores de ejecución

Ejemplo: multiplicación matriz/vector

Versión serial:

1 for (i = 0; i < m; i++) {
2     y[i] = 0.0;
3     for (j = 0; j < n; j++)
4         y[i] += A[i][j]*x[j];
5 }

Típicamente uno no escribe así en C; hace falta mapear arreglos 2D a arreglos 1D.

Supongamos que cada tarea es el ciclo interior:

1 for (j = 0; j < n; j++)
2     y[i] += A[i*n + j] * x[j];
